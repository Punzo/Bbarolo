Building from the source



Requisites


Barolo compiles and runs on Unix machines. It needs:

- C++ compiler, like the GNU compiler (better if > 4.2).
- CFITSIO library.
- FFTW library.
- WCS library.
- QT toolkit (> 4.0), only you would like to use the GUI.
- Gnuplot and Python (> 2.6) with the Astropy package.

Most of these libraries and packages should already be installed on scientific machines. Otherwise, you can easily install them through the terminal commands of the various package managers, i.e. apt-get on Ubuntu-based, pacman on Arch-based, yum on RPM-based distros, brew or port on Mac OS X. Note that the QT toolkit is only needed to compile the GUI, which is not mandatory. Gnuplot and Python are not needed to successfully compile the code, but without them Barolo will not produce any outputs.



Installation



1) Download either the .zip or the .tar source file.

2) Uncompress it (unzip *filename*.zip   or   tar -xvzf *filename*.tar) and enter the Barolo directory in a terminal prompt.

3) Launch the autoconfigure script (./configure). If the script is not executable: chmod +x configure. If there are any errors, follow the suggestions given by the script for manually set the path of libraries.

4) Compile the source by typing   make.

5) Optional: Compile the GUI by typing   make gui

6) Optional: Clean up the unnecessary files (make clean).
